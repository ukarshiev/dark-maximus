# План

1. Запустить `npx --yes nx build` последовательно для `bot`, `web-interface`, `user-docs`, `admin-docs`, соблюдая лучшие практики и при необходимости обращаясь к context7 для проверки актуальных настроек.
2. Для каждого приложения выполнить `npx --yes nx logs <project> --tail=200 --no-color 2>&1 | Select-String -Pattern 'error|exception|traceback|failed' -Context 2,0 -Wait -NoEmphasis`, анализируя вывод в соответствии с best practices и сверяясь с контекстом из context7.
3. Провести комплексный аудит кода и конфигураций на предмет ошибок, блокирующих функциональность, используя context7 как первоисточник и дополняя проверку официальной документацией.
4. По итогам сообщить результаты проверок и аудита, предложить дальнейшие действия (например, запуск тестов при необходимости).
