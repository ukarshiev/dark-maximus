# Приведение страницы настроек к адаптивному Material UI

## Цели

- Убрать остатки двухколоночной сетки (`settings-layout`), чтобы контент заполнял ширину.
- Настроить единый контейнер и отступы по канонам Material UI (8px шаг, `max-width`, `gap`).
- Сделать сетки (серверы, платежи и т.д.) адаптивными: 2 колонки >= 1024px, 1 колонка < 1024px.

## Изменения

1. **Обновить базовую разметку CSS** – файл `static/css/style.css`

- Перевести `.settings-layout` на `display: block`, удалить стили для отсутствующей `settings-sidebar`.
- Настроить `.settings-content`/`.settings-container`/`.settings-section` в духе Material UI: `max-width`, `padding`, `gap`, `border-radius`, `box-shadow`, `background`.
- Добавить responsive rules: desktop ≥1280px — `max-width` ~1200px и «карточки»-секции в две колонки при необходимости, медиазапросы 1024/768/480.

2. **Адаптивные сетки конкретных разделов** – `style.css`

- `#payments-tab`: удалить `width: clamp`, заменить `payment-systems-grid` на `grid-template-columns: repeat(auto-fit, minmax(320px, 1fr))`, задать `gap`, `align-items`.
- Обновить `.payment-system-card`, `.payment-column` стили (отступы, `box-shadow`, `background`).
- Аналогично для `.servers-grid` и карточек хостов (использовать flex/grid с авто-fit).

3. **Косметика/поддержка шрифтов и элементов** – `style.css`

- Выровнять поддерживающие элементы (`form-group`, `content-field-wrapper`, кнопки) под новые отступы.
- Проверить медиазапросы, убрать конфликтующие правила (например старые `repeat(2, 700px)` и `min-width`).

4. **Валидация верстки**

- В браузере проверить разделы «Панель», «Бот», «Платежные системы», «База данных», «Бекапы», «Справочники» (desktop + мобильное ~375px).
- Убедиться, что горизонтальный скролл отсутствует, активные подразделы работают.
