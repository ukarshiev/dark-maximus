# План правок install.sh

1. Обновить шаблон docker-compose

- Удалить строку `version: '3.8'`.
- Синхронизировать блоки `bot`, `docs`, `codex-docs` с текущим `docker-compose.yml` (volume-маппинги, healthcheck).
- Добавить секцию `user-cabinet`, идентичную определению в фактическом `docker-compose.yml` (контекст `./apps/user-cabinet`, порты, volumes, healthcheck).

2. Перепроверить шаг ожидания контейнеров

- Убедиться, что шаг проверки `timeout … nc -z 127.0.0.1 3003` и вызов логов соответствуют новому блоку сервиса.

3. Локальная проверка

- Запустить `install.sh` на тестовой машине (без применения) или выполнить локальную генерацию docker-compose через скрипт, чтобы убедиться, что итоговый файл совпадает с ожидаемым шаблоном.
