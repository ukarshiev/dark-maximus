# План обновления структуры документации

## Этапы

1. Проанализировать текущие разделы `docs/`, их назначение и количество.
2. Сгруппировать материалы в ≤10 верхнеуровневых каталогов по Diátaxis и доменам.
3. Предложить вложенную структуру каталогов и навигации, учитывая существующее содержимое.
4. Описать HTML-структуру раскрывающихся пунктов навигации (`md-nav__item--nested`).
