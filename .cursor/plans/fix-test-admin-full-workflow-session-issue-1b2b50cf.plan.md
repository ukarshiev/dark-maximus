# План исправления теста test_admin_full_workflow

## Проблема

Приложение использует `flask_session` с `SESSION_TYPE = 'filesystem'`, что может вызывать проблемы с `session_transaction()` в тестах, так как сессии хранятся в файлах, а не в памяти.

## Решение

Переопределить конфигурацию сессий в фикстуре `flask_app` для использования cookie-based сессий (по умолчанию Flask) вместо файловых сессий в тестах.

## Изменения

### 1. Обновить фикстуру `flask_app` в `tests/integration/test_web_panel/test_admin_workflow.py`

- Добавить переопределение `SESSION_TYPE = None` или удалить инициализацию `flask_session` для тестов
- Это позволит Flask использовать cookie-based сессии по умолчанию, которые корректно работают с `session_transaction()`
