# План миграции документации на MkDocs Material

1. Инвентаризация контента

- провести ревизию каталога `docs/`, классифицировать материалы по типам Diátaxis (tutorials/guides/reference/explanations) и зафиксировать, что нужно конвертировать в Markdown.

2. Подготовка структуры и форматов

- для каждого раздела собрать список файлов и, при необходимости, конвертировать HTML или нестандартные форматы в Markdown, добавив метаданные (дата, версия) по действующим правилам.

3. Настройка сборки MkDocs Material

- создать `mkdocs.yml`, задать многоуровневое меню (nav) с использованием возможностей Material (см. `/squidfunk/mkdocs-material`), подключить нужные плагины и настройки (navigation.sections, navigation.indexes и т. д.).

4. Развёртывание на http://localhost:3001/

- интегрировать MkDocs в текущий контейнер/сервис, обновить инструкции по запуску, проверить локально структуру и навигацию, затем передать итоговую схему пользователю.
