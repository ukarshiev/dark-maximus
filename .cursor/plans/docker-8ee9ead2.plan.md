# План устранения проблем со сборкой Docker

## Диагностика текущих прокси-настроек

- Проверить переменные окружения PowerShell/Windows: `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, `ALL_PROXY`.
- Изучить настройки Docker Desktop (Resources → Proxies) и файл `~/.docker/config.json` на предмет прокси.
- Просмотреть проектные файлы (`docker-compose.yml`, `.env`, `Dockerfile.*`) на наличие принудительных прокси-переменных.

## Коррекция конфигурации

- Удалить/исправить лишние прокси-переменные или настроить реальный прокси-сервис на `127.0.0.1:2080`.
- При необходимости скорректировать проектные конфигурации, исключив фиктивные прокси из Docker BuildKit или compose.
- Перезапустить Docker Desktop, чтобы новые настройки вступили в силу.

## Проверка и повторная сборка

- Повторно выполнить `docker compose up -d --build` и убедиться, что образы успешно скачиваются.
- Зафиксировать изменения: обновить `CHANGELOG.md` (с учётом правила вставки записи сверху) и синхронизировать версию в `pyproject.toml`.
- Подготовить ответ пользователю о необходимости пуша в GitHub.

## Контрольные проверки

- Убедиться, что в дальнейшем команды `npx nx` работают без дополнительных прокси-настроек.
- Проверить, что другие сервисы (бот, docs) стартуют корректно и не требуют ручных правок.
